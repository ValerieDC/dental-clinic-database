# Dental Clinic Database Design

## Project Overview
This project presents the design of a relational database for a dental clinic system. It models real-world operations including patients, dentists, appointments, treatments, and clinical history.

## Objectives
- Design a relational database based on real-world requirements
- Model patient care, treatments, and appointments
- Implement SQL structure and relationships
- Ensure data integrity and logical consistency

## Main Entities
- Patients
- Dentists
- Appointments
- Treatments
- Clinical History
- Locations
- Equipment

## Tools Used
- SQL
- ER Modeling

## Key Features
- Complex relational model
- Real-world constraints
- Clinical history tracking
- Appointment and treatment management

## Files Included
- ER diagram
- SQL scripts
